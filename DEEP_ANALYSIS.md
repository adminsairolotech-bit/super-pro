## 3) Synthesis Blueprint for `super-pro`

### What to take from each repo

- From `everything-claude-code`: core `.agents/skills/<skill>/` architecture and contribution taxonomy.
- From `cloude-ai-agiant-superpowers`: spec-plan-execute-review command workflow + cross-harness plugin support.

- From `second-brain-skills`: progressive disclosure model + PPTX and MCP utilities.
- From `HowToHunt`: security testing playbooks integrated as “secure-delivery skills” (referenced, not blindly merged due to GPL).
- From `multi-ai-system_prompts_leaks`: provenance-aware research corpus model (only with clean licensing and ethical boundaries).

### Ideal stack

- **Monorepo**
  - Node.js/TypeScript for CLI/orchestration and plugin packaging.
  - Python for data/analysis utilities (MCP client, corpus processing, pptx tooling).
- **Tooling**
  - pnpm or npm workspaces
  - Ruff + pytest (Python)
  - ESLint + Vitest (TS)
  - pre-commit + markdownlint + link checker
  - GitHub Actions CI matrix

### Recommended structure

```txt
super-pro/
  apps/
    cli/                      # TS CLI for skill orchestration
    docs-site/                # Optional docs portal
  packages/
    agent-core/               # runtime primitives
    skill-sdk/                # skill schema + validators
    plugin-bridges/           # claude/cursor/opencode/codex adapters
  skills/
    engineering/
    security/
    design/
    content/
    research/
  commands/
    write-plan.md
    execute-plan.md
    review-plan.md
  corpora/

    security-playbooks/       # curated references + normalized metadata
  schemas/
    skill.schema.json
    corpus-entry.schema.json
  scripts/
    readme-generators/
    quality-gates/
  .github/workflows/
  CONTRIBUTING.md
  GOVERNANCE.md
  LICENSE
```

### Key dependencies

- TS: `zod`, `commander`, `chalk`, `fast-glob`, `yaml`
- Python: `pydantic`, `typer`, `rich`, `python-pptx`, `mcp` sdk
- QA: `markdownlint-cli`, `lychee` (links), `pre-commit`
